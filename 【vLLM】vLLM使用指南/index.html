1. 概述与架构

1.1 什么是 vLLM

vLLM 是一个高性能的大型语言模型推理引擎，专为生产环境设计。核心特性：

  • PagedAttention：分页 KV 缓存管理，消除内存碎片，支持近乎无限的上下文长度
  • 连续批处理 (Continuous Batching)：动态合并请求，最大化 GPU 利用率
  • OpenAI 兼容 API：直接替换 OpenAI SDK 端点
  • 量化支持：AWQ、GPTQ、FP8、bitsandbytes、GGUF、MXFP4 等 20+ 种量化方法
  • 分布式推理：张量并行 (TP)、流水线并行 (PP)、数据并行 (DP)、专家并行 (EP)
  • KV 缓存传输：支持 Prefill/Decode 分离部署，对接 LMCache、Mooncake 等外部缓存系统
  • 多模态：视觉、音频、视频模型推理
  • 推测解码：Eagle、Medusa、MTP 等多种推测解码方法

8.5 前缀缓存 (Prefix Caching)

前缀缓存的工作原理：

  1. 调度器将 prompt 按 KV 块大小 (默认 16 tokens) 分块
  2. 计算每个块的哈希值 -> 查找是否已缓存
  3. 命中 -> 重用 KV 块，跳过该块的 prefill 计算
  4. 未命中 -> 执行 prefill 并缓存结果

8.6 Chunked Prefill

分块 prefill 将长 prompt 切分为多个小块，与 decode 请求交错执行，避免长 prefill 阻塞短请求的响应。

9. 分布式推理

9.1 张量并行 (Tensor Parallelism, TP)

将模型的每一层切分到多个 GPU 上。

使用规则：

  • TP 大小必须能被注意力头数整除
  • 对于 MoE 模型，EP 可以与 TP 结合使用
  • 推荐 GPU 拓扑优化 (单节点 > NVLink > PCIe)

9.2 流水线并行 (Pipeline Parallelism, PP)

将模型的不同层分配到不同 GPU。

9.3 数据并行 (Data Parallelism, DP)

多个模型副本同时处理不同请求。

DP 模式：

  • external_lb: 外部负载均衡 (K8s Pod 模式)，每个 --data-parallel-rank 一个实例
  • hybrid_lb: 混合负载均衡，节点内 vLLM LB + 外部 LB 跨节点均衡
  • 本地模式: 单个 vLLM 实例内 DP

9.4 上下文并行 (Context Parallelism, CP)

将长上下文切分到多个 GPU。

9.5 专家并行 (Expert Parallelism, EP)

专为 MoE 模型设计，将不同专家放到不同 GPU。

13.3 性能调优检查清单

  1. KV 缓存: 使用 FP8 KV Cache (--kv-cache-dtype fp8) 可节省~50% KV 缓存内存
  2. 前缀缓存: 默认已启用 (enable_prefix_caching=True)，确保未被禁用
  3. CUDA Graph: 默认 O2 级别已启用，确认未被 --enforce-eager 禁用
  4. Chunked Prefill: 默认启用，可通过 --max-num-batched-tokens 调优
  5. MoE 模型: 启用 EP + DBO，选择合适的 All2All 后端
  6. TP 配置: 确保 TP 大小能被注意力头数整除
  7. 编译缓存: 保持 VLLM_USE_AOT_COMPILE 启用 (默认)
  8. FlashInfer: 确保使用 FlashInfer 后端 (默认优先)
