1. 概述与架构

1.1 什么是 Mooncake

Mooncake 是一个高性能分布式 KVCache 传输与存储系统，专为 LLM 推理场景设计。它提供了：

  • Transfer Engine（传输引擎）: 多协议数据平面，支持 RDMA、TCP、NVLink、CXL、NVMe-oF 等
  • Mooncake Store（分布式存储）: 基于 Master-Client 架构的分布式 KV 缓存池
  • EP（Expert Parallelism）: MoE 模型的专家并行 CUDA 扩展
  • PG（Process Group）: PyTorch 分布式通信后端
  • P2P Store: 去中心化对象共享（Go 实现）

1.2 核心组件架构

1.3 组件关系

组件 作用 依赖
Transfer Engine 底层多协议数据传输（C++） RDMA/TCP 硬件
Mooncake Store Master 管理分布式段、KV 元数据、副本、HA Transfer Engine
Mooncake Store Client 数据存取代理（put/get/transfer） Master
MooncakeDistributedStore Python 高级 API（tensor 存取、段管理） Store Master/Client
MooncakeConnector vLLM V1 PD 分离插件 Transfer Engine
EP Extension MoE 专家并行 CUDA kernel CUDA/PyTorch
PG Extension PyTorch ProcessGroup 后端 Transfer Engine

6. HTTP/REST API 参考

6.1 Master 管理 API（端口 9003）

所有端点挂载在 Master 的 metrics HTTP 服务器上（由 --metrics_port 控制，默认 9003）。

方法 端点 描述
GET /health 健康检查，返回 JSON {"status": "ok"}
GET /metrics Prometheus 格式指标 (text/plain)
GET /metrics/summary 人类可读的指标摘要
GET /role 当前 HA 角色 (LEADER/FOLLOWER)
GET /ha_status HA 运行时状态
GET /leader Leader 信息 (JSON)
GET /get_all_keys 列出所有已存储的 key
GET /get_all_segments 列出所有段
GET /query_key?key=<key> 查询某个 key 的副本信息
GET /batch_query_keys?keys=k1,k2,... 批量查询 key 副本列表
GET /query_segment?segment=<name> 查询段使用情况 (used/capacity)

6.1.1 Drain Job API

方法 端点 描述
POST /api/v1/drain_jobs 创建 drain job（迁移数据出段）
GET /api/v1/drain_jobs/query?job_id=<id> 查询 drain job 状态
POST /api/v1/drain_jobs/cancel?job_id=<id> 取消 drain job
GET /api/v1/segments/status?segment=<name> 查询段 drain 状态
