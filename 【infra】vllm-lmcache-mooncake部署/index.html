Mooncake坏境准备

简介

Mooncake 是一个高性能分布式 KVCache 传输与存储系统，专为 LLM 推理场景设计。所以Mooncake只是一个KVCache管理工具。主要功能特性有：

  • Transfer Engine（传输引擎）: 多协议数据平面，支持 RDMA、TCP、NVLink、CXL、NVMe-oF 等
  • Mooncake Store（分布式存储）: 基于 Master-Client 架构的分布式 KV 缓存池
  • EP（Expert Parallelism）: MoE 模型的专家并行 CUDA 扩展
  • PG（Process Group）: PyTorch 分布式通信后端
  • P2P Store: 去中心化对象共享（Go 实现）

Mooncake源码安装

  1. 下载github上Mooncake源码

  2. 运行自动检测 OS 并安装所有依赖的脚本 dependencies.sh现在这个脚本支持大部分操作系统

    如果使用手动下载可以使用如下命令：

  3. CMake构建，提供了很多构建选项：

    选项 默认值 描述
    WITH_TE ON 构建 Transfer Engine
    WITH_STORE ON 构建 Mooncake Store
    WITH_STORE_GO OFF 构建 Store Go 绑定
    WITH_P2P_STORE OFF 构建 P2P Store
    WITH_RUST_EXAMPLE OFF 构建 Rust Transfer Engine 接口
    WITH_STORE_RUST ON 构建 Store Rust 绑定
    WITH_EP OFF 构建 Expert Parallelism (需要 CUDA)
    USE_NOF OFF 启用 NVMe over Fabric SSD 池
    STORE_USE_ETCD OFF 使用 etcd 后端
    STORE_USE_REDIS OFF 使用 Redis 后端
    STORE_USE_K8S_LEASE OFF 使用 K8s Lease 选主
    STORE_USE_JEMALLOC OFF Master 使用 jemalloc
    USE_CUDA OFF 启用 CUDA 支持
    USE_ASCEND OFF 启用华为 Ascend NPU 支持
    USE_HIP OFF 启用 AMD HIP 支持
    USE_MUSA OFF 启用 Moore Threads MUSA 支持

  4. 如果是普通的Nvidia生态的普通服务器，那么直接最小依赖安装。

​ 这种方式编译，最终得到的mooncake只支持纯 CPU 传输引擎，也就是说数据一定是卸载到内存，然后通过cpu管理将这部分数据传输到共享内存池或者SSD等各个介质中。

如果要使用要使用CUDA、Ascend NPU的一些特性NVLink 跨节点传输、节点内 NVLink 传输就需要开启USE_CUDA等选项。

  1. 构建后，关键产物位于 build/ 目录。如果不使用python API，那么就可以直接执行c++可执行程序。

    产物 路径
    mooncake_master build/mooncake-store/src/mooncake_master

    store.so build/mooncake-integration/store/store.so

  2. 如果要使用python接口，就需要Wheel 安装一下。Mooncake自带了构建脚本scripts/build_wheel.sh。

    在执行scripts/build_wheel.sh可以通过坏境变量来管理python接口的内容。

    环境变量 包名
    （默认） mooncake-transfer-engine
    NON_CUDA_BUILD=1 mooncake-transfer-engine-non-cuda
    CU13_BUILD=1 mooncake-transfer-engine-cuda13
    NPU_BUILD=1 mooncake-transfer-engine-npu

    安装后，以下 CLI 命令可用：

    • mooncake_master
    • mooncake_client
    • transfer_engine_bench
    • mooncake_http_metadata_server
    • mc_store_rest_server
    • transfer_engine_topology_dump

    高可用 (HA):

    Flag 默认值 描述
    --enable_ha false 启用 HA 模式
    --ha_backend_type "etcd" HA 后端: etcd / redis / k8s
    --ha_backend_connstring "" HA 后端连接字符串
    --etcd_endpoints "" etcd 端点（分号分隔）
    --cluster_id "mooncake_cluster" 集群 ID
    --root_fs_dir "" 存储后端根目录

    存储与内存:

    Flag 默认值 描述
    --memory_allocator "offset" 内存分配器: cachelib / offset
    --allocation_strategy "random" 分配策略: random / free_ratio_first / cxl
    --quota_bytes 0 存储配额（0=默认 90% 容量）
    --global_file_segment_size INT64_MAX 全局文件段大小

    Offload（磁盘卸载）:

    Flag 默认值 描述
    --enable_offload false 启用磁盘卸载
    --offload_on_evict false 驱逐时才执行 LOCAK_DISK 卸载
    --offload_force_evict false 超过 offload 上限时强制驱逐
    --promotion_on_hit false 访问时从磁盘提升到内存
    --promotion_admission_threshold 2 提升准入阈值（1=关闭二次访问门控）
    --promotion_queue_limit 50000 最大提升并发数

    快照:

    Flag 默认值 描述
    --snapshot_backup_dir "" 快照本地备份目录

    --snapshot_object_store_type "" 对象存储类型: local / s3

    Flag 默认值 描述

    HTTP 元数据服务器:

    Flag 默认值 描述
